对抗样本防御主要分为两大类策略：输入变换(Input Transformation) 和对抗训练(Adversarial Training)

本文基于 LeNet-5 在 MNIST 手写数字识别认为上的实验，系统对比了两种输入变换方法（中值滤波、位深度缩减）和两种对抗防御策略（Two-step 和 Iterative）的效果。

<!--more-->



## 输入变换

在推理阶段对输入图像进行预处理，削弱对抗扰动的影响。

优点是无需重新训练模型，计算开销小，适用于已部署逆向的快速防御。

### 中值滤波

中值滤波通过滑动窗口取中值的方式平滑图像，能有效去除孤立噪声。

```python
def median_smooth(imgs, kernel_size=3):
    """
    基于手动循环滑窗的中值滤波实现
    imgs: [B, C, H, W] tensor
    kernel_size: 必须为奇数 (如 3, 5)
    """
    B, C, H, W = imgs.shape
    pad = kernel_size // 2
    smoothed = torch.zeros_like(imgs)
    for i in range(pad, H - pad):
        for j in range(pad, W - pad):
            roi = imgs[:, :, i - pad : i + pad + 1, j - pad : j + pad + 1]  # roi 的形状为 [B, C, kernel_size, kernel_size]
            # 提取roi的中值 ，形状为 [B, C]
            median = roi.reshape(B, C, -1).median(dim=2)[0]
            # 将中值赋值给smoothed[:, :, i, j]
            smoothed[:, :, i, j] = median
    return smoothed
```

### 位深度缩减

将像素值映射到 `[0, 2^depth - 1]` 的整数范围，四舍五入后再映射回 `[0, 1]`。本质上是量化操作，能抹平小于量化步长的扰动。

例如 `depth=1` 时只有 0/1 两种取值（二值化），`depth=4` 时有 16 种取值

```python
def bit_depth_reduce(imgs, depth=4):
    """
    对输入图像进行位深度缩减
    imgs: [B, C, H, W] tensor, 值域 [0, 1]
    depth: 目标位深度
    return: 位深度缩减后的图像张量
    """
    max_val = 2 ** depth - 1
    # 映射到 [0, max_val]
    reduced = imgs * max_val
    # 四舍五入取整
    reduced = torch.round(reduced)
    # 映射回 [0, 1]
    reduced = (reduced / max_val).clamp(0, 1)

    return reduced
```

`evaluate()` 评估模型的准确率

```python
def evaluate(imgs, labels, model):
    # 用model预测imgs，并得到预测标签pred_label
    model.eval()
    pred = model(imgs)
    pred_label = pred.argmax(1)

    # 计算预测标签与真实标签的匹配数目
    correct_cnt = (pred_label == labels).sum().item()

    print(f'match rate: {correct_cnt/labels.shape[0]}')
    return pred_label
```

**中值滤波**的效果很不理想，对 clean 样本有损伤，对 FGSM 和 PGD 防御几乎为 0，甚至恶化攻击。根本原因是 MNIST 手写数字识别依赖局部笔画细节，中值滤波去除扰动同时也抹掉了这些关键局部特征。kernel_size 越大，局部细节丢失越严重，整体准确率反而下降。

**位深度缩减**：深度为 1、2 时效果很好，而当深度为3、4时效果又很差，这与量化步长和干扰强度有关。

实验使用 epsilon = 0.15

- 深度为 2 时，量化步长为 1/3  > 0.15

- 深度为 3 时，量化步长为 1/7 < 0.15

总的来说量化步长大于干扰强度时，位深度缩减能去掉干扰（四舍五入），而代价是也会失去一些细节，但这种细节是全局的。见下面可视化效果。

## 对抗训练

在训练阶段将对抗样本纳入训练过程，增强模型自身对扰动的抗干扰能力。优点是防御更彻底、可泛化，但训练成本较高。

### Two step

先一次性生成全部对抗样本，再将这些样本作为输入更新模型

#### FGSM 

```python
def adv_train_two_step(data_loader, epoch, lr, criterion):
    model = LeNet5()
    model.load_state_dict(torch.load('model/lenet5.pt'))
    model.train()

    optimizer = torch.optim.Adam(model.parameters(), lr=lr)

    for e in range(epoch):
        t = tqdm(data_loader)
        for img, label in t:
            # 前向传播并在混合batch上计算分类损失
            pred = model(img)
            loss = criterion(pred, label)   
            # 反向传播
            loss.backward()
            # 更新模型参数
            optimizer.step()
            # 清空梯度
            optimizer.zero_grad()

            t.set_postfix(epoch=e, loss=loss.item())

    return model
```

### 迭代对抗训练

每次更新模型后重新生成对抗样本，让模型持续面对更新后的更强的对抗样本攻击

#### FGSM
```python
def adv_train_iter_fgsm(data_loader, epoch, lr, criterion, epsilon, adv_loss_weight=1.):
    model = LeNet5()
    model.load_state_dict(torch.load('model/lenet5.pt'))
    model.train()

    optimizer = torch.optim.Adam(model.parameters(), lr=lr)

    for e in range(epoch):
        t = tqdm(data_loader)
        for img, label in t:
            # 前向传播并对良性样本计算损失
            benign_loss = criterion(model(img), label)
            # 生成对抗样本，然后进行前向传播并计算对抗样本的损失
            adv_img = fgsm(img, epsilon, model, criterion, label)
            adv_loss = criterion(model(adv_img), label)
            # 计算总损失，然后反向传播
            loss = benign_loss + adv_loss_weight * adv_loss
            # 反向传播
            loss.backward()
            # 更新模型参数
            optimizer.step()
            # 清空梯度
            optimizer.zero_grad()
            t.set_postfix(epoch=e, benign_loss=benign_loss.item(), adv_loss=adv_loss.item())

    return model
```

```python
def adv_train_iter_pgd(data_loader, epoch, lr, criterion, epsilon, alpha, iter=5, adv_loss_weight=1.):
    model = LeNet5()
    model.load_state_dict(torch.load('model/lenet5.pt'))
    model.train()

    optimizer = torch.optim.Adam(model.parameters(), lr=lr)

    for e in range(epoch):
        t = tqdm(data_loader)
        for img, label in t:
            # 前向传播并对良性样本计算损失
            benign_loss = criterion(model(img), label)
            # 生成对抗样本，然后进行前向传播并计算对抗样本的损失
            adv_img = pgd(img, epsilon, alpha, iter, model, criterion, label)
            adv_loss = criterion(model(adv_img), label)
            # 计算总损失，然后反向传播
            loss = benign_loss + adv_loss_weight * adv_loss
            # 反向传播
            loss.backward()
            # 更新模型参数
            optimizer.step()
            # 清空梯度
            optimizer.zero_grad()

            t.set_postfix(epoch=e, benign_loss=benign_loss.item(), adv_loss=adv_loss.item())

    return model
```
