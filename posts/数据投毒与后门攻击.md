深度学习模型的安全不仅取决于推理阶段对抗样本的鲁棒性，也依赖训练数据与供应链的可信性。攻击者可通过数据投毒（Data Poisoning）或后门攻击（Backdoor Attack）操纵模型，使其在满足特定条件时出现错误输出，同时在普通样本上保持高精度，具备极强的隐蔽性与危害性。

<!--more-->

**对抗样本**：发生在部署后的推理阶段。攻击者对输入添加细微扰动，诱使模型以高置信度输出错误类别。

**数据投毒**：在数据收集与预处理阶段实施攻击，攻击者通过向训练数据中注入精心构造的恶意样本，破坏模型的学习过程，从而降低模型整体性能或使其产生特定的错误行为。与对抗样本不同，数据投毒发生在模型训练之前，影响的是模型本身而非单次推理。

**后门攻击**：一种常见的数据投毒形式。攻击者在训练阶段注入带 trigger 的样本并赋予 target label，令模型学习到触发器到目标标签的捷径。并在无触发器的干净输入上模型仍保持较高准确率。

干净标签（clean-label）与脏标签（dirty-label）：前者保持样本的语义标签不变，隐蔽性更强；后者将带触发器样本标注为目标类，更易实现但更易被发现。



## 攻击面与威胁模型

下面列出常见的攻击面：

- 代码投毒：在公共 DL 框架、训练脚本或依赖项中注入恶意逻辑
- 外包训练/代训：第三方可在训练过程中注入后门
- 预训练供应链：发布被植入后门的教师模型或权重（模型市场风险）
- 数据收集：污染公开数据集或数据管道
- 协作/联邦学习：控制部分客户端实施局部数据投毒；
- 部署后篡改：通过故障注入或内存攻击修改权重（越权/入侵范畴）

易受攻击场景：

- 直接采用第三方数据集、第三方平台训练或第三方预训练模型。
- 复杂数据流水线、标注外包、弱审计流程。



## 常见后门攻击方法

| 方法 | 核心思想 |
|------|---------|
| BadNets | 在图像上粘贴可见的触发器补丁 |
| Blended Attack | 将触发器图案以低透明度混合到原图中，更加隐蔽 |
| Label-Consistent Backdoor Attacks | 保持投毒样本标签一致，绕过人工审查 |
| Hidden Trigger Backdoor Attacks | 隐藏触发器，使投毒样本在视觉上与干净样本无异 |
| TrojanNet | 插入独立的木马模块 |

## 评测指标

| 指标                  | 说明                                                         |
| --------------------- | ------------------------------------------------------------ |
| 干净样本准确率（CDA） | 模型在未受污染的测试集上的准确率。后门攻击往往保持该数值较高以隐藏痕迹。 |
| 攻击成功率（ASR       | 在带触发器的测试集上模型输出目标类别的比例。有效后门通常追求高 ASR。 |
| 隐蔽性                | 触发器的可感知性（人眼可见性、统计可见性），可用像素扰动范数、SSIM/PSNR 或频域统计近似衡量。 |
| 鲁棒性与可迁移性      | 触发器在数据增强（裁剪、翻转、光照）或跨模型/跨任务上的触发能力。 |

**构建混合数据集**：按照指定的毒化率，将干净样本与投毒样本按比例混合。

```python
def create_poisoned_dataset(clean_imgs, clean_labels, poisoned_imgs, poisoned_labels, poison_rate=0.1):
    total_num = len(clean_imgs)  # 固定总样本数
    num_poison = int(total_num * poison_rate)  # 计算需要投毒的样本数量
    num_clean = total_num - num_poison

    # 随机从干净样本和投毒样本池中抽取指定数量的样本
    clean_idx = torch.randperm(len(clean_imgs))[:num_clean]
    poison_idx = torch.randperm(len(poisoned_imgs))[:num_poison]

    # 拼接成最终的混合数据集
    mixed_imgs = torch.cat([clean_imgs[clean_idx], poisoned_imgs[poison_idx]], dim=0)
    mixed_labels = torch.cat([clean_labels[clean_idx], poisoned_labels[poison_idx]], dim=0)

    return mixed_imgs, mixed_labels
```

```python
class CustomDataset(torch.utils.data.Dataset):
    def __init__(self, imgs, labels):
        self.imgs = imgs
        self.labels = labels

    def __len__(self):
        return len(self.imgs)

    def __getitem__(self, idx):
        return self.imgs[idx], self.labels[idx]
```

### Blended Attack 后门攻击

BadNets 的触发器是直接粘贴在图像上的可见补丁，容易被简单的人工审查或异常检测发现

Blended Attack 混合攻击使用更隐蔽的方式，将 trigger 以低透明度混合到原始图像中：

$$
x_{poisoned}=(1 - \alpha) \cdot x_{clean} + \alpha \cdot t
$$

t：trigger ，可以为随机噪声、特定纹理等

alpha 为混合比例，通常取较小值 0.1~0.2

```python
def add_trigger_blended(imgs, trigger_pattern=None, alpha=0.1):
    """
    Blended Attack 触发器注入
    将触发器图案以低透明度混合到原图中

    Args:
        imgs: 原始图像 (B, C, H, W)
        trigger_pattern: 触发器图案，与图像尺寸相同。若为 None 则使用随机噪声
        alpha: 混合比例，越小越隐蔽，但可能需要更高的毒化率才能成功植入后门
    
    Returns:
        poisoned_imgs: 投毒后的图像
    """
    poisoned_imgs = imgs.clone()

    if trigger_pattern is None:
        # 使用固定种子生成随机噪声作为触发器，确保所有样本使用同一触发器
        generator = torch.Generator()
        generator.manual_seed(42)
        trigger_pattern = torch.rand(imgs.shape[1:], generator=generator)

    # 混合：x_poisoned = (1 - alpha) * x_clean + alpha * trigger
    poisoned_imgs = (1 - alpha) * poisoned_imgs + alpha * trigger_pattern.unsqueeze(0)

    # 确保像素值在合法范围内
    poisoned_imgs = torch.clamp(poisoned_imgs, 0, 1)

    return poisoned_imgs
```

### Label-Consistent Backdoor Attacks

保持投毒样本标签一致，绕过人工审查

比如构造一批手写数字 0，标签也是 0，但是有 trigger 的有毒样本混入数据集

模型会将手写数字 0 和 trigger 一起学习作为 label = 0 的特征，面对其他手写数字，触发 trigger 从而错误识别为 0

此方法能有效绕过依赖标签一致性的人工审查。

### Hidden Trigger Backdoor Attacks  

隐藏触发器，使投毒样本在视觉上与干净样本无异

对 Label-Consistent Backdoor Attacks 进行改进，之前模型将手写数字 0 和 trigger 一起学习作为 label = 0 的特征，面对其他数字时不一定触发 trigger，因为数字不是 0

Hidden Trigger Backdoor Attacks 则是弱化模型学习手写数字 0 特征的能力，这样一看到 trigger 就认为是 0，而忽略真正的数字是什么

弱化模型学习手写数字 0 特征的方式，比如使用 PGD 生成对抗样本，再编码一个特定的模式（比如高斯噪声），作为训练数据集。这个特定的模式就是 trigger





## 后门攻击防御

1. 认证后门防御：基于随机滤波，但效果较差

2. 经验后门防御：触发器-后门，**样本携带触发器**、**模型包含后门**、**触发器与后门匹配**。防御策略围绕破坏这三个条件展开

如果以不同阶段/层面看：

- 数据层面：严格做数据来源审计、检查标签-特征异常、检查可疑一致性
- 训练阶段：使用强数据增强、使用鲁棒训练策略、数据净化后再训练
- 模型层面：Fine-Pruning（微调+剪枝）、反向检查后门特征、模型蒸馏等
- 推理阶段：多视图一致性检测、输入预处理、输入异常检测

本次实验，对使用 Blended Attack 后门攻击的中毒模型，trigger 透明度 alpha = 0.1，毒化率为 0.1（训练数据集有毒样本比例）

此时干净样本准确率为 0.9431，trigger 触发成功率为 0.9760



使用两种防御策略

1. 基于预测差异的数据清洗
2. 模型层面的微调减枝



### 预测差异检测中毒样本

干净模型与理想的中毒模型，对干净样本的预测行为基本一致，但对于包含 trigger 的中毒样本，预测结果产生显著分歧。

- 中毒模型会因为 trigger 激活后门，输出攻击者设定的标签
- 干净模型不受 trigger 影响，根据样本真实语义进行预测

比较模型对同一批混合数据的预测差异，识别出被投毒的样本

```python
def create_poisoned_dataset_with_trace(clean_imgs, clean_labels, poisoned_imgs, poisoned_labels, poison_rate=0.1):
    total_num = len(clean_imgs)
    num_poison = int(total_num * poison_rate)
    num_clean = total_num - num_poison

    clean_idx = torch.randperm(len(clean_imgs))[:num_clean]
    poison_idx = torch.randperm(len(poisoned_imgs))[:num_poison]

    mixed_imgs = torch.cat([clean_imgs[clean_idx], poisoned_imgs[poison_idx]], dim=0)
    mixed_labels = torch.cat([clean_labels[clean_idx], poisoned_labels[poison_idx]], dim=0)

    is_poisoned = np.array([False] * num_clean + [True] * num_poison)

    shuffle_idx = torch.randperm(len(mixed_imgs))
    mixed_imgs = mixed_imgs[shuffle_idx]
    mixed_labels = mixed_labels[shuffle_idx]
    is_poisoned = is_poisoned[shuffle_idx.numpy()]

    return mixed_imgs, mixed_labels, is_poisoned, num_clean, num_poison
```

计算预测差异：让干净模型和中毒模型对混合数据集进行预测，记录：

- 预测不一致的样本
- 高置信度差异的样本：预测标签相同，但是对预测的置信度差异极大的样本

```python
def detect_poisoned_samples_prediction_disagreement(mixed_imgs, mixed_labels, clean_model, poisoned_model, poison_rate=0.1):
    clean_model.eval()
    poisoned_model.eval()

    dataset = CustomDataset(mixed_imgs, mixed_labels)
    loader = torch.utils.data.DataLoader(dataset, batch_size=128, shuffle=False)

    clean_preds = []
    poisoned_preds = []
    confidences = []

    with torch.no_grad():
        for imgs, _ in tqdm(loader, desc='计算预测差异'):
            # 获取两个模型的预测结果
            clean_out = clean_model(imgs)
            poisoned_out = poisoned_model(imgs)

            clean_pred = clean_out.argmax(dim=1)       # 干净模型的预测标签
            poisoned_pred = poisoned_out.argmax(dim=1) # 中毒模型的预测标签
            
			 # 计算两个模型对各自预测结果的置信度
            clean_conf = torch.softmax(clean_out, dim=1).max(dim=1)[0]
            poisoned_conf = torch.softmax(poisoned_out, dim=1).max(dim=1)[0]

            clean_preds.append(clean_pred)
            poisoned_preds.append(poisoned_pred)
            
            # 记录置信度的绝对差异
            confidences.append(torch.abs(clean_conf - poisoned_conf))
	# 合并所有批次的预测结果
    clean_preds = torch.cat(clean_preds).numpy()
    poisoned_preds = torch.cat(poisoned_preds).numpy()
    confidences = torch.cat(confidences).numpy()
    
	# 判断条件1：两个模型的预测标签是否不一致
    disagreement_mask = (clean_preds != poisoned_preds)

	# 判断条件2：置信度差异是否超过阈值（这里取差异最大的前 N 个样本）
    # 假设中毒样本数量约为 poison_rate * 总样本数，稍微放宽阈值（乘以1.5倍）
    num_poison = int(len(mixed_imgs) * poison_rate * 1.5)
    confidence_threshold = np.sort(confidences)[-num_poison] # 取差异最大的前 N 个样本

    high_conf_mask = (confidences >= confidence_threshold)
	
    # 综合两个条件：预测不一致 或 置信度差异极大 的样本，被视为可疑中毒样本
    poisoned_indices = np.where(np.logical_or(disagreement_mask, high_conf_mask))[0]
	
    print(f'预测不一致的样本数: {np.sum(disagreement_mask)}')
    print(f'高置信度差异的样本数: {np.sum(high_conf_mask)}')

    return poisoned_indices
```

### 微调与剪枝 Fine-Pruning

Fine-Pruning是一种不依赖数据检测的模型修复技术，结合**微调（Fine-tuning）** 和**剪枝（Pruning）** 

- 微调：使用少量干净数据对中毒模型进行高学习率微调，帮助模型遗忘触发器关联；
- 剪枝：在干净数据上统计各层神经元/通道的平均激活值，剪除对干净样本激活较低（但可能对触发器响应强烈）的神经元；

- 恢复微调：低学习率再次微调，恢复主任务性能。

注册 PyTorch 的前向钩子（hook），收集模型各层在处理一批干净数据时的输出激活值，并计算每个神经元（对于全连接层）或通道（对于卷积层）的平均激活强度。

```python
def get_activation_statistics(model, dataloader):
    model.eval()
    activation_stats = {} # 用于存储各层的激活值
    
    hooks = []  # 存储钩子函数，以便后续移除
    def hook_fn(name):
        def hook(module, input, output):
            if name not in activation_stats:
                activation_stats[name] = []
            # 将输出激活值保存到列表中，并转移到CPU
            activation_stats[name].append(output.detach().cpu())
        return hook
    
    # 为所有卷积层和全连接层注册 hook
    for name, module in model.named_modules():
        if isinstance(module, nn.Conv2d) or isinstance(module, nn.Linear):
            # 注册前向传播钩子，当该模块执行时调用 hook_fn 函数
            hooks.append(module.register_forward_hook(hook_fn(name)))
    
    # 前向传播，收集激活值
    with torch.no_grad():
        for imgs, _ in dataloader:
            model(imgs)
    
    # 移除 hook
    for hook in hooks:
        hook.remove()
    
    # 计算平均激活值
    avg_activations = {}
    for name, acts in activation_stats.items():
        acts = torch.cat(acts, dim=0)   # 拼接所有 batch 的数据
        if len(acts.shape) == 4:  # Conv2d: (N, C, H, W)
            # 在批次、高度和宽度维度上平均，得到每个通道的平均激活值
            avg_act = acts.mean(dim=(0, 2, 3)) 
        else:  # Linear: (N, F)
            # 在批次维度上平均，得到每个神经元的平均激活值
            avg_act = acts.mean(dim=0)  
        avg_activations[name] = avg_act
    
    return avg_activations

# 创建用于统计的数据加载器
# 使用前 10000 个干净样本进行统计
stat_loader = torch.utils.data.DataLoader(CustomDataset(clean_train_img[:10000], clean_train_label[:10000]), batch_size=128, shuffle=False) # 不随机打乱，确保结果可复现

# 计算微调后模型中各层的平均激活值
avg_activations = get_activation_statistics(fine_pruned_model, stat_loader)
```

根据上一步计算出的平均激活值，对不同的网络层（如`conv1`, `fc1`等）按预设比例（`prune_ratios`）进行剪枝。

如对 `fc1` 层剪枝50%，即意味着将该层中平均激活值最低的 50% 的神经元的权重置零。

代码通过创建掩码（mask）并将其应用于对应层的权重和偏置来实现。

```python
# 不同层使用不同的剪枝比例
prune_ratios = {
    'conv1': 0.2,
    'conv2': 0.3,
    'fc1': 0.5,
    'fc2': 0.5,
    'fc3': 0.3
}

# 遍历所有有平均激活值的层
for layer_name in avg_activations.keys():
    # 跳过没有在剪枝比例字典中定义的层
    if layer_name not in prune_ratios:
        continue

    prune_ratio = prune_ratios[layer_name]       # 获取当前层的剪枝比例
    avg_act = avg_activations[layer_name]        # 获取当前层的平均激活值
    num_prune = int(len(avg_act) * prune_ratio)  # 计算需要剪枝的神经元数量
    
    if num_prune == 0:
        continue
    
    # 计算剪枝的阈值：
    # 1. 对平均激活值进行排序
    # 2. 取第 num_prune 个值作为阈值
    # 激活值低于此阈值的神经元将被剪枝
    threshold = torch.sort(avg_act)[0][num_prune]
    
    # 创建 mask
    # 1. 比较每个神经元的平均激活值与阈值
    # 2. 激活值大于阈值的神经元保留（值为1）
    # 3. 激活值小于等于阈值的神经元剪枝（值为0）
    mask = (avg_act > threshold).float()
    
    # 在模型中查找对应层并应用剪枝掩码
    for name, module in fine_pruned_model.named_modules():
        if name == layer_name:
            if isinstance(module, nn.Conv2d):   # 如果是卷积层
                # 将不需要的通道权重置零
                module.weight.data *= mask.view(-1, 1, 1, 1)
                if module.bias is not None:
                    module.bias.data *= mask
            elif isinstance(module, nn.Linear): # 如果是全连接层
                module.weight.data *= mask.unsqueeze(1)
                # 将不需要的神经元权重置零
                if module.bias is not None:
                    module.bias.data *= mask
            print(f'已对层 {name} 剪枝 {num_prune}/{len(mask)} 个神经元')
```

剪枝操作可能会损害模型在主要任务上的性能，再次使用干净数据对剪枝后的模型进行低学习率的微调，以恢复其分类准确率。
