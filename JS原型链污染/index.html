Javascript 原型链污染

得益于 JavaScript 极高的灵活性，以及强大的继承机制，JS 存在一种独特的安全问题：原型链污染。

JS 将所有内容视为对象，可以从每个对象出发，可以通过原型链，一直向上访问父类属性，直到 Object 原型对象，一但在这里修改或添加属性，能够影响所有的对象。

我将介绍 JS对象 如何通过原型链继承，根据这一特性，污染原型链，并分析一个简单的CVE

通过原型链继承

很简单的示例，子类 Dog 继承自父类 Animal，并各自有一些属性

创建一个Dog实例 a，调用实例 a 的 sleep 方法

显然 这个方法 继承自父类，JavaScript 是如何完成继承的呢？通过原型链。

介绍原型链之前，先介绍什么是原型对象

只有 函数/类 有 prototype，作为新实例的原型模板

  1. 对于静态方法，可以直接使用 类名称.方法名 调用，但不适用于实例方法

  2. 实例方法定义在原型对象（ prototype）上，如果想直接调用类的实例方法，必须使用类名称.prototype.方法名

    或者使用实例名.方法名

所有实例共享同一方法（节省内存），也就是共享方法

现在介绍 __proto__，所有 对象（含实例）都有 __proto__，指向其构造函数的prototype

每个对象通过这个特殊方法，可以一层层向上获取父类的 prototype，也就是原型链，实现继承

对象类型 __proto__ prototype
普通对象 有 无
函数/类 有 有
函数实例 有 无
基本类型值 无（但包装对象有） 无

这个机制是 JavaScript 实现继承和共享方法的核心。
