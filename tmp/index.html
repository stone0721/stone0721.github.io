  • 主要逻辑：程序接受 16 字节 hex 输入作为初始状态 s[32] ~ s[35]，然后通过 22 轮计算更新状态 s。
  • 交互机制：
    • 启动 Python 脚本 “saw” 模块，传递端口，Python 连接到 C#。
    • C# 从 Python 接收动态 .NET 程序集，加载 Curse 类, 执行 Do方法 更新 state s 和 sp (程序计数器) 。
    • 同时, C# 连接到 Java 的 JDWP 端口, 获取类/方法 ID, 设定断点, 调用 Hack 类中的方法 ( hcm 列表中的方法) , 获取 long 值更新 s。
    • Crack 函数向 Python 发送固定字节数组, 接收返回数据作为 Java 方法参数。
    • Horror 函数使用 AES 解密 Java 返回的数据, 然后 re base64 并发送。
  • 加密部分：使用 AES-256-CBC, key = SHA256(assembly full name), iv = from s[1017:1033] 低字节 (前6个0, 后10个具体值)。
  • 输出：经过 22 轮后, 输出 s[32] ~ s[35] 的低 32 位 hex (32 字符)。
  • flag.enc：这个 48 字节 hex 可能是用 AES 加密的 flag, 但使用代码中 key/iv 解密后 padding 无效, 可能 iv 动态或 assembly name 是 “polyencrypt” 而非 “saw” , 或需要运行程序计算出正确 key/iv, 或 the enc is the output of the program with the flag as input, need to reverse the rounds to find the flag.

程序本质是一个多态加密系统, Python “saw” 模块负责发送 assembly 和处理 Crack 请求, Java 负责一些计算函数.
